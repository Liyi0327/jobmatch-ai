# JobMatch AI：AI 求职岗位匹配与求职管理系统

JobMatch AI 是一个面向 AI 应用开发实习求职场景的轻量级求职工作台。项目支持上传 PDF / Word 简历、导入岗位 JD、调用大模型生成岗位匹配分析，并通过岗位库和求职看板管理投递进度。

项目重点不是单纯生成一段简历建议，而是尝试把「简历解析、岗位理解、匹配分析、求职进度管理」整合成一个完整的 AI 应用工作流。

---

## 一、项目背景

在实习求职过程中，求职者通常需要反复完成以下操作：

- 阅读大量岗位 JD；
- 判断岗位是否适合自己；
- 针对岗位修改简历；
- 准备可能被问到的面试问题；
- 记录岗位投递状态；
- 跟进笔试、面试、Offer 等流程。

这些工作分散在招聘网站、文档、表格和聊天工具中，管理成本较高。

JobMatch AI 希望通过一个轻量级 AI 工作台，帮助求职者更高效地完成岗位筛选、简历匹配分析和求职进度管理。

---

## 二、核心功能

### 1. 简历上传与解析

支持上传 PDF 或 Word 格式简历，系统会自动解析简历文本，并将其作为后续岗位分析的默认简历。

主要功能包括：

- 上传 PDF / DOCX 简历；
- 自动提取简历文本；
- 查看当前简历解析内容；
- 保留历史简历记录；
- 支持切换当前使用简历；
- 支持移除当前简历或删除历史简历。

---

### 2. 岗位 JD 导入与结构化

用户可以将招聘平台中的岗位 JD 粘贴到系统中，系统会调用大模型对 JD 进行结构化解析。

解析内容包括：

- 岗位名称；
- 公司名称；
- 城市；
- 岗位职责；
- 岗位要求；
- 加分项；
- 岗位关键词；
- 投递优先级建议。

---

### 3. 简历与岗位匹配分析

系统基于当前简历和岗位 JD，调用大模型生成结构化分析结果。

分析内容包括：

- 综合匹配度；
- 匹配等级；
- 总体结论；
- 岗位关键词；
- 简历关键词；
- 候选人优势；
- 能力短板；
- 简历优化建议；
- 推荐补充项目；
- 模拟面试问题；
- 面试表达建议。

---

### 4. 岗位库管理

所有导入过的岗位会保存在岗位库中，方便统一管理。

岗位库支持：

- 查看所有岗位；
- 搜索岗位；
- 按状态筛选岗位；
- 设置当前岗位；
- 更新岗位状态；
- 添加岗位备注；
- 删除岗位；
- 查看原始 JD；
- 对指定岗位重新进行匹配分析。

---

### 5. 求职看板

求职看板用于展示当前求职进度，不再堆叠大量表格，而是聚焦最关键的信息。

看板展示内容包括：

- 岗位总数；
- 有效跟进岗位数；
- 待处理岗位数；
- Offer 数量；
- 已进入投递流程的岗位；
- 每个岗位当前处于哪一步；
- 当前最应该处理的岗位；
- 每个岗位的下一步行动建议。

---

### 6. 演示模式

为了避免在作品集展示中暴露真实个人信息，项目提供演示模式。

演示模式支持：

- 一键加载虚拟简历；
- 一键加载虚拟岗位；
- 一键加载虚拟分析结果；
- 一键清空演示数据。

演示数据均为虚拟内容，不包含真实个人隐私，可用于截图、录屏和面试展示。

## 三、技术栈

| 模块 | 技术 |
|---|---|
| 前端界面 | Streamlit |
| 后端逻辑 | Python |
| 数据存储 | SQLite |
| 简历解析 | PyMuPDF、python-docx |
| 大模型调用 | OpenAI SDK 兼容接口 |
| 数据处理 | Pandas |
| 环境管理 | Conda / venv |
| 版本管理 | Git |
